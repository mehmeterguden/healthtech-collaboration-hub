Demo Requirements
User Guide–Based Evaluation



Course	SENG 384 – Spring 2026
Evaluation Type	Demo (Offline Video)
Duration	Maximum 5 minutes per group
Basis	User Guide 
 
 
1. Overview
This document defines the requirements for the video demonstration of your HEALTH AI platform. The demo is based directly on your User Guide.
2. Requirements
You should demonstrate the following scenarios. 
 
Scenario 1: Registration & Login    [~45 sec]
→ User Guide Section: 2 (Getting Started)

1. Show the registration page. Attempt registration with a non-.edu email → show error.
2. Register a new account with a valid .edu email. Show the role selection step.
3. Show the email verification step (or explain if mocked).
4. Log in with the new account. Show the Dashboard.
 
Scenario 2: Post Creation & Management    [~1 min]
→ User Guide Section: 3 (Post Management)

1. Create a new post: fill in ALL form fields (title, domain, expertise, city, description, etc.).
2. Save as Draft first. Show it appears with “Draft” status.
3. Publish the post. Show it changes to “Active” status.
4. Edit the post (change one field). Save and show the update.
5. Show the full post list with pre-loaded seed data visible.
 
Scenario 3: Search & Filtering    [~30 sec]
→ User Guide Section: 4 (Search & Partner Discovery)

1. Use at least 2 different filters (domain, city, status, or expertise).
2. Show that results update correctly based on filters.
3. Clear filters and show all posts again.
 
Scenario 4: Meeting Request Workflow    [~1 min]
→ User Guide Section: 5 (Meeting Request Process)

1. Switch to a different user account (e.g., healthcare professional).
2. Browse posts and open a post detail page.
3. Click “Express Interest” / “Request Meeting”.
4. Show the NDA acceptance step (if applicable).
5. Propose a time slot.
6. Switch back to the post owner and accept/confirm the meeting.
7. Show the post status changes to “Meeting Scheduled”.
 
Scenario 5: Admin Panel    [~1 min]
→ User Guide Section: 7 (Admin Panel)

1. Log in as Admin.
2. Show the user management screen (list users, filter by role).
3. Show the post management screen (view posts, remove one inappropriate post).
4. Show the activity logs (filter by date or action type).
5. Export logs as CSV (click the export button, show the download).
 
Scenario 6: Profile & GDPR    [~30 sec]
→ User Guide Sections: 6, 8 (Profile Management, Notifications)

1. Show the profile edit page.
2. Show the account deletion flow (navigate to it, show the warning — do NOT actually delete).
3. Show the data export option (if implemented).
4. Show at least one notification in the system.
3. Evaluation Criteria
Evaluation Item	Criteria
S1: Registration & Login	.edu validation, role selection, email verification, successful login
S2: Post Creation & Mgmt	All form fields, Draft/Active lifecycle, edit works, seed data visible
S3: Search & Filtering	Multiple filters work, results update correctly
S4: Meeting Request	Interest, NDA, time slots, accept/confirm, status update
S5: Admin Panel	User mgmt, post mgmt, logs, CSV export
S6: Profile & GDPR	Profile edit, delete account flow, data export, notifications
